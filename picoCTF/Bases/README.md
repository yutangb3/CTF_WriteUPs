基数変換するにしても62進数以上は必要である。<br>

これはすべてのデータを英数字で表すエンコード方式のものである。<br>
実際に変換するとフラグが得られる。<br>

## ここから学んだこと
base64とは基数を生かした変換方式の一種である。<br>
昔は文字専用だった。つまりバイナリデータ7bitを送っていた。<br>
ところが画像としてバイナリデータを送るとコンピューター側としては<br>
当然文字であると解釈し始める。<br>
だがしかし前提として画像のバイナリデータである上に<br>
コンピューターによっては改行の合図と間違えて解釈してしまい<br>
データが壊れてしまう。そこで画像を画像のバイナリデータとして<br>
送れないならデータを文字化して<br>
受け取り側は文字化されたものを画像のバイナリデータとして変換する。<br>
こうすることでデータが壊れず処理できる。<br>
**変換アルゴリズム**<br>
1.二進数へ変換<br>
2.6bitずつ区切る。<br>
3.区切って余りのbitがでたら0を追加する。<br>
4.2の6乗、つまり**64通り**表せるので変換表を用いて英数字に直す。<br>
